Contracts define what operators and integrators can rely on between releases.

This section is reserved for:

- Behavior contracts for `ottoctl` and service lifecycle commands.
- Data and schema contracts (including OpenAPI-generated references).
- Compatibility policy for versioned docs and release tags.
- Security and authentication boundaries for live docs capabilities.

Contract pages should use explicit version context and avoid undocumented
implied behavior.
