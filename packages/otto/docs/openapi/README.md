# Otto OpenAPI

Otto publishes autogenerated OpenAPI specs from Fastify route code.

## What this documents

- External API (`/external/*`): runtime operations intended for operators/control-plane clients.
- Internal API (`/internal/*`): local tool integration endpoints used by Otto runtime tooling.

## What this does not guarantee

- Internal API is not a public compatibility contract.
- OpenAPI docs do not replace operational runbooks or architecture docs.
- Background one-shot creation does not stream live output; status is polled by session id.

## Generate specs

Run from repository root:

```bash
pnpm -C packages/otto run docs:openapi:generate
```

Generated files:

- `packages/otto/docs/openapi/external.v1.json`
- `packages/otto/docs/openapi/internal.v1.json`

## Served docs

- External UI: `/external/docs`
- External JSON: `/external/openapi.json`
- Internal UI: `/internal/docs`
- Internal JSON: `/internal/openapi.json`

Both UI and JSON docs routes are intentionally served without token auth for discovery and integration.
